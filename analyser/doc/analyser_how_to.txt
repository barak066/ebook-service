#summary HOWTO по использованию Analyser

= Что делает Analyser =

Analyser предназначен для обхода сайтов и сохранения информации о найденных книгах в базе данных

= HOWTO по использованию Analyser =

==Как запустить сканер по сайту? ==

Находясь в корне папки analyser, введите:
{{{
$python manage.py site_scanner
}}}

будет выведен на экран примерно такой список сайтов, доступных для сканера:
{{{
following sites are available to scan:
 * http://lib.ru
 * http://magazines.russ.ru
 * http://nehudlit.ru
 * http://data.fbreader.org
 * http://vse-knigi.org
 * http://flibusta.net
 * http://manybooks.net
 * http://smashwords.com
 * http://www.feedbooks.com
}}}

Далее, напишите:
{{{
$python manage.py site_scanner http://lib.ru
}}}
и сканер начнет сканировать сайт lib.ru.
Сканер в любой момент можно остановить, нажав комбинацию клавиш *Ctrl+C*. После этого, если его запустить заново, сканирование продолжится с того места, где сканер остановился.

== Перезапуск ==

Так как сканер продолжает работу с того места, где остановился, а иногда (например, если найденные книги были удалены) необходимо запускать сканер заново. Для этого напишите:
{{{
$python manage.py site_scanner --rerun http://vse-knigi.org
}}}
Сканер заново начнет сканировать указанный сайт.

== Обновление ==
Вообще, обновить информацию с сайта можно, просто запустив сканер заново. (В таком случае существующие книги просто будут обновлены, а новые добавлены).

В данный момент, только для lib.ru есть альтернативный, более быстрый,  метод обновления. Чтобы им воспользоваться, напишите:
{{{
$python manage.py site_scanner --refresh http://lib.ru
}}}
Программа проверит, какие ссылки сайта нуждаются в обновлении, и запустит сканер по этим ссылкам. При этом, если вы прервете работу обновления, то запустить заново нужно той же командой (с параметром *--refresh*)

== Также == 
Сканер сайтов можно запускать находясь в любой директории (не обязательно его рабочей).
